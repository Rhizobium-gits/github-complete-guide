# GitHub完全ガイド — はじめてのバージョン管理からチーム開発まで

> 「コードを書いたことはあるけれど、Gitって何？」という方から、チームで本格的に開発を始めたい方まで。本書を一冊読み通せば、GitとGitHubの全体像が手に取るように分かります。

## 本書の特徴

プログラミングの入門書を読み終え、いざ自分のコードを管理しよう、誰かと一緒に開発しようと思ったとき、多くの人が最初にぶつかる壁が「Git」と「GitHub」です。ターミナルに打ち込むコマンド、聞き慣れない英語の用語、ブランチやマージといった抽象的な概念——これらが絡み合って、とっつきにくい印象を与えてしまいます。

本書では、そうした壁をひとつずつ丁寧に取り除くことを目指しました。

- **用語の意味を最初に説明します。** 「コミット」「プッシュ」「プルリクエスト」など、Git特有の言葉が出てくるたびに、その語源や背景を含めてかみ砕いて解説します。
- **なぜその操作が必要なのかを語ります。** コマンドの羅列ではなく、「どんな場面で」「なぜ」その操作を行うのかという文脈を重視しています。
- **実際の画面と手を動かす例を示します。** `<!-- screenshot: 説明 -->` の箇所にはスクリーンショットを追加できるようになっており、実際の画面を見ながら学習できます。
- **段階的に進みます。** 第1部の基礎知識から順に読み進めれば、無理なくステップアップできる構成になっています。

## 対象読者

- プログラミングを始めたばかりで、コードの管理方法を知りたい方
- 研究室やチームで共同作業をするためにGitHubを導入したい方
- 自己流でGitを使っているが、体系的に学び直したい方
- ポートフォリオとしてGitHubを活用したい学生・若手エンジニア

## スクリーンショットの追加方法

本書のなかで `<!-- screenshot: 説明 -->` と記載されている箇所は、スクリーンショットの挿入位置を示すプレースホルダーです。以下の手順で画像を追加できます。

1. 該当する画面のスクリーンショットを撮影します
2. `docs/images/` フォルダに画像ファイルを保存します（例: `docs/images/github-signup.png`）
3. プレースホルダーを画像タグに置き換えます: `![説明](images/ファイル名.png)`
4. 変更をコミットしてプッシュします
