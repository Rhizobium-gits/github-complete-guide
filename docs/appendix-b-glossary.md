# 付録B: 用語集

## A

### Add (アド)
ファイルの変更をステージングエリアに追加する操作。`git add`。

### Amend (アメンド)
直前のコミットを修正する操作。`git commit --amend`。

### Artifact (アーティファクト)
GitHub Actionsのワークフロー内で生成・保存されるファイル。ビルド成果物やテスト結果など。

### Assignee (アサイニー)
IssueやPRの担当者。

## B

### Base branch (ベースブランチ)
PRのマージ先となるブランチ。通常は `main` や `develop`。

### Blame (ブレイム)
ファイルの各行がいつ・誰によって変更されたかを表示する機能。`git blame`。

### Blob (ブロブ)
Gitが管理するファイルの内容データ。GitHub URLの `/blob/` はファイル表示を意味する。

### Branch (ブランチ)
開発の分岐。並行して独立した作業ができる。コミットの系列に付けたポインタ。

## C

### Cherry-pick (チェリーピック)
特定のコミットだけを現在のブランチに適用する操作。

### CI/CD (シーアイ/シーディー)
Continuous Integration / Continuous Deployment。コードの統合・テスト・デプロイを自動化する手法。

### Clone (クローン)
リモートリポジトリの完全なコピーをローカルに作成する操作。`git clone`。

### CODEOWNERS (コードオーナーズ)
ファイルやディレクトリの責任者を定義するファイル。PRで自動的にレビュアーに追加される。

### Collaborator (コラボレーター)
リポジトリへの書き込み権限を持つ共同作業者。

### Commit (コミット)
ファイルの変更を履歴に記録する操作・単位。スナップショットとメタデータ（著者・日時・メッセージ）で構成。

### Conflict (コンフリクト)
2つのブランチで同じ部分が異なる変更をされた場合に、自動マージできない状態。

### Contributor (コントリビューター)
プロジェクトに貢献した人。コードの提供、Issue報告、ドキュメント改善など。

## D

### Dependabot (ディペンダボット)
依存関係の脆弱性を自動検出し、更新PRを作成するGitHubの機能。

### Deploy (デプロイ)
アプリケーションを本番環境に配置・公開すること。

### Diff (ディフ)
ファイルやコミット間の差分。追加行（緑）と削除行（赤）で表示される。

### Draft PR (ドラフトPR)
まだレビュー準備ができていないPR。マージボタンが無効化される。

## F

### Fast-forward (ファストフォワード)
マージ時に新しいコミットを作らず、ブランチポインタを前に進めるだけのマージ方式。

### Fetch (フェッチ)
リモートの最新情報を取得する操作。ローカルのファイルは変更されない。`git fetch`。

### Fork (フォーク)
他のユーザーのリポジトリを自分のアカウントにコピーする操作。OSSへの貢献の第一歩。

## G

### Git
分散型バージョン管理システム。Linus Torvaldsが2005年に開発。

### GitHub
Gitリポジトリのホスティングサービス。Microsoft傘下。

### GitHub Actions
GitHub内蔵のCI/CDプラットフォーム。YAMLファイルでワークフローを定義。

### GitHub Pages
GitHubリポジトリから静的Webサイトをホスティングする無料サービス。

### .gitignore (ギットイグノア)
Gitに追跡させたくないファイルやフォルダを指定するファイル。

### GPG (ジーピージー)
GNU Privacy Guard。コミットの署名に使われる暗号化ツール。

## H

### Hash (ハッシュ)
コミットを識別する一意のID。SHA-1で生成される40文字の16進数文字列。例: `a1b2c3d4e5f6...`。

### HEAD (ヘッド)
現在チェックアウトしているコミットやブランチを指すポインタ。

### Hook (フック)
特定のGitイベント（コミット前後など）で自動実行されるスクリプト。`.git/hooks/` に配置。

## I

### Index (インデックス)
ステージングエリアの別名。次のコミットに含める変更を保持する場所。

### Issue (イシュー)
バグ報告、機能要望、タスク管理のためのチケットシステム。

## L

### Label (ラベル)
IssueやPRに付けるカテゴリタグ。`bug`、`enhancement` など。

### LFS (エルエフエス)
Git Large File Storage。大きなファイルを効率的に管理する拡張機能。

### License (ライセンス)
ソフトウェアの利用・改変・配布の条件を定めた法的文書。MIT, Apache 2.0, GPL等。

## M

### Main (メイン)
デフォルトブランチの名前。以前は `master` が使われていた。

### Markdown (マークダウン)
テキストに簡単な記法で書式を付けられる軽量マークアップ言語。GitHubで広く使用。

### Merge (マージ)
あるブランチの変更を別のブランチに統合する操作。

### Milestone (マイルストーン)
Issueをリリースや期限でグループ化する機能。進捗がパーセンテージで表示される。

## O

### Open Source (オープンソース)
ソースコードが公開され、誰でも利用・改変・配布できるソフトウェア。

### Organization (オーガニゼーション)
チームでリポジトリを管理するためのGitHubアカウント。権限管理やチーム分けが可能。

### Origin (オリジン)
クローン元のリモートリポジトリに付けられるデフォルトの名前。

## P

### PAT (パット)
Personal Access Token。HTTPSでGitHubに認証するためのトークン。

### PR / Pull Request (プルリクエスト)
ブランチの変更をメインブランチに取り込むことを提案する機能。コードレビューの場。

### Pull (プル)
リモートの変更をローカルに取り込む操作。`fetch` + `merge`。

### Push (プッシュ)
ローカルのコミットをリモートリポジトリに送信する操作。

## R

### Raw (ロー)
ファイルの生データ。`raw.githubusercontent.com` で取得可能。

### Rebase (リベース)
ブランチの分岐点を移動させる操作。コミット履歴が直線的になる。

### Reflog (リフログ)
HEADの移動履歴。`reset --hard` で消したコミットも追跡できる。

### Release (リリース)
タグに加えてリリースノートやバイナリを含む、ソフトウェアの公開単位。

### Remote (リモート)
ネットワーク上の別のリポジトリ。GitHubのリポジトリなど。

### Repository (リポジトリ)
プロジェクトのファイル群と変更履歴を保存する場所。repoとも略される。

### Reset (リセット)
コミットやステージングを取り消す操作。`--soft`、`--mixed`、`--hard` の3モード。

### Revert (リバート)
指定したコミットを打ち消す新しいコミットを作成する操作。安全な取り消し方法。

### Review (レビュー)
PRの変更内容をチェックしてフィードバックを提供するプロセス。

### Runner (ランナー)
GitHub Actionsのワークフローを実行するサーバー（仮想マシン）。

## S

### Secret (シークレット)
GitHub Actionsで使用するAPIキーやパスワードなどの秘密情報。暗号化して保管される。

### SHA (エスエイチエー)
Secure Hash Algorithm。Gitがコミットを識別するのに使用するハッシュアルゴリズム。

### SSH (エスエスエイチ)
Secure Shell。暗号化された通信プロトコル。GitHubとの認証に使用。

### Squash (スカッシュ)
複数のコミットを1つにまとめる操作。PRマージ時によく使われる。

### Stage / Staging (ステージ/ステージング)
コミットに含めるファイルの変更を選択する操作・領域。

### Stash (スタッシュ)
未コミットの変更を一時的に退避させる機能。

### Submodule (サブモジュール)
リポジトリ内に別のリポジトリを埋め込む機能。

## T

### Tag (タグ)
特定のコミットに付ける名前。主にバージョン番号に使用。

### Tree (ツリー)
Gitが管理するディレクトリ構造。GitHub URLの `/tree/` はフォルダ表示を意味する。

## U

### Upstream (アップストリーム)
Fork元のリポジトリ。または、ローカルブランチが追跡しているリモートブランチ。

## V

### Verified (ベリファイド)
GPG署名されたコミットに表示されるバッジ。コミットの作者が本人であることを証明。

## W

### Webhook (ウェブフック)
特定のイベント発生時に外部URLへHTTPリクエストを自動送信する仕組み。

### Wiki (ウィキ)
リポジトリに付属するドキュメントページ。Markdownで記述可能。

### Workflow (ワークフロー)
GitHub Actionsで定義する自動化処理の設定ファイル（YAML形式）。

### Working Directory / Working Tree (ワーキングディレクトリ/ツリー)
実際にファイルを編集する場所。`.git` フォルダを除いたプロジェクトのファイル群。
